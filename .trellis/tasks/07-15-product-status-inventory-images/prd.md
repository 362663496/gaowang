# 商品状态管理与库存图片

## Goal

让操作员能在商品列表直接启用、禁用和删除商品，并在当前库存表中通过缩略图快速识别商品；完成验证后发布到 `ssh aliyun`。

## Background

- 商品模型已有 `Enabled` 和 `ImagePath` 字段；商品 API 的原有列表入口位于 `apps/api/internal/http/handlers/products.go:26`（模型见 `apps/api/internal/models/models.go:64`）。
- 商品列表表格及其状态操作位于 `apps/web/src/app/(app)/products/page.tsx:172`。
- 当前库存接口已经预加载完整 `Product`，因此响应中已有 `ImagePath`；库存页直接渲染该字段，无需新增图片查询或数据表字段（`apps/api/internal/http/handlers/inventory.go:34`、`apps/web/src/app/(app)/inventory/page.tsx:75`）。
- `InventorySnapshot` 与 `StockMovement` 对商品均使用 `OnDelete:RESTRICT`，现有数据库契约禁止删除有库存快照或流水历史的商品（`apps/api/internal/models/models.go:83`、`apps/api/internal/models/models.go:93`）。
- 线上实际使用 `/opt/gaowang/releases/<timestamp>` 版本目录和 `/opt/gaowang/current` 原子软链；`gaowang-api.service`、`gaowang-web.service` 分别监听 `127.0.0.1:9509`、`127.0.0.1:9510`，Nginx 从 `:9508` 转发。PostgreSQL 卷、上传图片和备份位于独立共享位置，不随版本目录切换。

## Requirements

1. 商品列表的每一行提供启用/禁用操作，操作成功后页面状态和已启用数量同步更新。
2. 启用状态使用现有成功色，禁用状态使用与启用明显不同的现有状态色；不能只依赖颜色，仍显示“启用”或“禁用”文字。
3. 商品列表的每一行提供删除操作，并在执行前要求用户确认。
4. 状态变更和删除通过后端接口持久化，并写入审计日志。
5. 删除不存在的商品返回明确的未找到错误；删除被库存快照或流水引用的商品不得破坏库存和历史数据，并向页面返回可理解的冲突提示。
6. 当前库存表在商品名称前展示该商品的缩略图；没有图片时展示与商品页一致含义的占位图标。
7. 沿用现有商品管理权限，不引入新的角色或权限层级。
8. 完成本地后端测试、前端 lint/type-check/test/build，再部署到 `ssh aliyun`，保留现有数据库与上传文件并验证线上健康状态和页面可访问性。
9. 商品禁用仅改变并展示 `Enabled` 状态；本次不改变禁用商品参与入库、出库或库存调整的现有行为。

## Acceptance Criteria

- [x] 点击已启用商品的“禁用”操作后，后端 `Enabled` 变为 `false`，列表显示带禁用色的“禁用”，已启用计数减一。
- [x] 点击已禁用商品的“启用”操作后，后端 `Enabled` 变为 `true`，列表显示带成功色的“启用”，已启用计数加一。
- [x] 删除操作有确认步骤；未被库存或流水引用的商品可删除并从列表消失。
- [x] 有库存快照或流水历史的商品删除失败时，商品、库存和历史流水保持不变，页面显示“请禁用商品”的明确提示。
- [x] 状态变更与删除产生对应审计记录。
- [x] 当前库存每行在商品名称前显示 40px 左右的商品缩略图；无图商品显示占位图标，名称和编码仍可读。
- [x] 不新增数据库字段，既有商品、库存、流水、上传图片和数据库卷在发布后保持完整。
- [x] 禁用商品仍沿用当前库存操作规则，不因本次改动被额外过滤或拦截。
- [x] 本地质量命令全部通过，阿里云上的健康接口和 Web 页面在部署后可访问。

## Out of Scope

- 批量启用、禁用或删除。
- 编辑或替换既有商品图片。
- 级联删除库存快照、出入库流水或财务历史。
- 为库存缩略图新增图片预览弹窗。
- 改变禁用商品的入库、出库或库存调整资格。
