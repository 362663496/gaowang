# 完善商品库存操作体验

## Goal

完善商品与库存操作：修复删除逻辑，增加分页、可搜索商品下拉和商品修改，并提升低库存查看、入库店铺记录及流水商品识别效率；在此基础上一次性把全站前端迁移到 Ant Design，验证通过后发布到 `ssh aliyun`。

## Background

- 商品页已正确发出 `DELETE /products/:id` 并处理 `204`；后端当前会拒绝任何已有库存快照或流水的商品（`apps/web/src/app/(app)/products/page.tsx:68`、`apps/web/src/lib/api.ts:36`、`apps/api/internal/http/handlers/products.go:87`）。
- 线上日志显示未使用商品删除返回 `204`，而同一已使用商品在 2026-07-16 20:33 连续三次返回 `409`。根因是现有删除保护与新的归档删除预期不一致，不是按钮未发送请求。
- 当前库存按商品全局汇总，`InventorySnapshot` 只有 `ProductID` 主键；销售出库选择店铺也只给流水标记店铺，不代表门店独立库存（`apps/api/internal/models/models.go:68`、`apps/api/internal/services/inventory.go:96`）。

## Requirements

### R1. 商品删除与归档

- 从未产生库存快照或流水的商品继续硬删除，同时清理其上传图片。
- 已有库存快照或流水、且当前库存为 0 的商品执行归档删除：从商品列表、新库存操作选择器、当前库存、库存品类和低库存运营数据中隐藏，历史流水继续保留。
- 当前库存不为 0 时拒绝删除并返回明确原因；前端必须醒目显示服务端原始错误信息，不刷新页面、不移除商品。
- 归档商品继续占用原编码；本次不提供归档列表、恢复入口或编码复用。
- 归档商品仍可在流水历史筛选中搜索和选择，并标注“已归档”。历史流水、仪表盘最近流水、销售额、成本、毛利和商品排行继续保留；商品排行标注“已归档”。

### R2. 低库存明细

- 当前库存页的“低库存”数字可点击，点击后在当前页把下方库存表筛选为低库存商品。
- 提供清晰的“显示全部”入口恢复完整表格。
- 口径沿用 `LowStockThreshold > 0 && Quantity <= LowStockThreshold`。

### R3. 商品搜索筛选

- 入库、销售出库、库存调整和流水筛选四个入口统一使用一个可输入搜索的商品下拉框，不再并排显示独立搜索框。
- 搜索按商品名称或编码进行不区分大小写的模糊匹配；清空搜索恢复完整可选范围。
- 新库存操作不出现归档商品；流水筛选包含并标注归档商品。

### R4. 入库店铺

- 入库时可选店铺，默认不选，店铺非必填。
- 选择店铺时把 `ShopID` 保存到该条入库流水；未选择时保存为空。
- 当前库存继续按商品全局汇总，不增加门店库存维度。

### R5. 流水商品图片

- 仅在“流水记录”页面商品列显示约 40px 的商品缩略图。
- 无图片时使用与当前库存页一致的占位图标；商品名称和编码仍可读。
- 本次不改仪表盘“最近流水”，不增加图片预览。

### R6. 验证与发布

- 完成后端测试与静态检查，以及前端 lint、类型检查、测试和生产构建。
- 覆盖归档/硬删除/有库存拒绝、历史财务不变、可选入库店铺、搜索筛选和页面交互的回归验证。
- 发布到 `ssh aliyun` 的新版本目录，保留共享数据库、上传和备份目录；原子切换版本并保留旧版本回滚点。

### R7. 列表分页

- 商品、店铺、当前库存、流水记录、操作记录和用户列表使用服务端分页，默认每页 20 条。
- 分页响应统一返回当前页、每页数量、总条数和总页数；筛选条件变化时回到第一页。
- 商品/店铺/用户选择器以及仪表盘、报表所需完整数据可显式请求 `all=true`，不把选项或汇总误截断为第一页。
- 仪表盘摘要、报表趋势/排行和“最近一次备份”不是管理列表，不新增分页。

### R8. 商品修改

- 商品列表增加“修改”入口，复用商品表单修改名称、编码、默认进货价、默认销售价、低库存阈值和备注。
- 可选上传新图片替换原图片；未选择文件时保留现有图片。本次不增加单独删除图片功能。
- 修改成功后刷新当前页并记录 `product.update` 审计；服务端错误在表单内原样展示。

### R9. Ant Design 全站升级

- 登录、应用外壳及全部后台页面一次性迁移到 Ant Design 6，使用框架自带的布局、导航、表格、分页、表单、弹窗、搜索选择、上传、图片、通知和状态组件。
- 保留现有业务流程、API 契约、权限可见性和中文文案，不在迁移中改变库存或财务口径。
- 所有管理列表统一使用 Ant Design `Table` 的受控服务端分页；商品、店铺、用户等选择器统一使用可搜索 `Select`，不再保留原生 `datalist` 或旁置搜索框。
- 服务端错误必须在对应表单、页面或通知中显示原始消息；删除等危险操作使用框架确认弹窗，不使用浏览器原生确认框。
- 迁移完成后删除旧 Radix/Tailwind UI primitives、旧消息/分页/状态组件及不再使用的依赖，避免两套 UI 基础设施长期并存。
- 使用 Ant Design 官方 Next.js App Router 样式注册方案，统一中文 locale、主题 token、紧凑型后台布局和响应式移动导航。

## Technical Notes

- 当前库存页的低库存数字只是静态汇总；仪表盘和报表已有相同口径的明细实现可参考（`apps/web/src/app/(app)/inventory/page.tsx:63`、`apps/web/src/app/(app)/dashboard/page.tsx:108`、`apps/web/src/app/(app)/reports/page.tsx:68`）。
- 三个库存表单共用 `ProductSelect`，流水筛选单独使用原生下拉，共四个入口（`apps/web/src/features/inventory/action-forms.tsx:196`、`apps/web/src/app/(app)/stock-movements/page.tsx:65`）。
- `StockMovement.ShopID` 已可空，流水接口已预加载 `Shop`；入库增加店铺不需要新表（`apps/api/internal/models/models.go:79`、`apps/api/internal/http/handlers/movements.go:16`）。
- 流水接口已预加载完整 `Product`，前端类型已有 `ImagePath`；流水图片不需要新接口（`apps/web/src/features/types.ts:37`）。
- 线上采用 `/opt/gaowang/releases/<timestamp>` 与 `/opt/gaowang/current` 软链，API/Web systemd 服务和共享数据目录沿用现有发布拓扑。
- 前端采用 Ant Design 6.5.1 与 `@ant-design/nextjs-registry`；React 19 无需 v5 兼容补丁，App Router 根布局负责注入首屏样式。

## Out of Scope

- 门店独立库存、门店间调拨及按店铺计算低库存。
- 级联删除库存快照、流水或财务历史。
- 归档商品列表、恢复、相同编码复用和批量归档。
- 新低库存页面或弹窗。
- 仪表盘流水图片和图片预览。
- 单独删除商品图片、批量修改商品或修改归档商品。
