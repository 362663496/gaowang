# 库存页面导出含图片

## Goal

在当前库存页面提供可下载、可编辑的 Excel 库存清单，并让每条记录携带对应商品图片，方便线下查看、盘点和流转。

## Background

- 当前页面位于 `apps/web/src/app/(app)/inventory/page.tsx`，按 20 条分页展示，并支持“全部库存 / 低库存”筛选。
- 库存接口已返回完整商品信息与 `Product.ImagePath`；商品图片由 API 上传目录保存，并通过同源 `/uploads/*` 提供。
- 现有图片类型为 JPG、PNG、WebP，单张最大 5 MB。
- 项目目前没有库存导出能力或已安装的表格导出依赖。

## Requirements

- R1：库存页面提供清晰可用的“导出 Excel”入口。
- R2：导出文件必须是 `.xlsx`，商品图片必须真实嵌入对应行，不能只写图片链接。
- R3：导出当前筛选条件下的全部记录，不受页面分页影响；默认导出全部库存，低库存筛选开启时仅导出全部低库存记录。
- R4：列顺序固定为：图片、商品名称、商品编码、数量、移动平均成本、库存金额、库存状态、更新时间。
- R5：单条记录没有图片、图片丢失或图片无法解析时，保留该库存行且不能导致整次导出失败。
- R6：保持现有库存查询、筛选、分页和库存操作行为不变。
- R7：导出能力沿用库存页面现有访问权限，管理员与员工均可使用。

## Acceptance Criteria

- [ ] AC1：用户点击导出后获得可由常用表格软件打开的 `.xlsx` 文件。
- [ ] AC2：工作表包含 R4 的全部字段且顺序一致；名称和编码为独立列，数量和金额可继续参与表格计算。
- [ ] AC3：有图商品的图片嵌入对应记录行；无图或坏图商品仍正常出现在文件中。
- [ ] AC4：普通状态、低库存筛选和不同分页下的导出范围符合 R3。
- [ ] AC5：现有库存功能未回归，后端与前端质量检查通过。

## Out of Scope

- CSV、PDF 或其他导出格式。
- 定时导出、自动发送和独立报表中心。
- 数据库结构或库存计算规则变更。
